3.3 Requisitos no funcionales

Código – Nombre Descripción
RNF01 - Intuitividad La interfaz del sistema debe ser intuitiva y fácil de usar, minimizando la curva de aprendizaje, especialmente para usuarios no familiarizados con sistemas informáticos
RNF02 - Personalización Los usuarios deben poder personalizar su interfaz y recibir información relevante según su rol y preferencias.
Confiabilidad
RNF03 - Disponibilidad El sistema debe estar disponible para los usuarios el 99.9% del tiempo, garantizando un acceso continuo y confiable a sus funcionalidades.
RNF04 - Tolerancia a Fallos El sistema debe ser capaz de detectar y recuperarse de errores o fallos de manera automática, minimizando el impacto en los usuarios y evitando la pérdida de datos.
Seguridad
RNF05 contraseña segura El acceso al sistema debe estar protegido mediante mecanismos de autenticación fuertes, como contraseñas seguras, tendrá una función la cual permitirá que no se visualice al ingresar la contraseña para mayor seguridad
Eficiencia
RNF06 - Capacidad: El sistema debe poder manejar un gran volumen de datos (horarios, asistencias, usuarios) sin afectar su rendimiento.
Portabilidad
RNF07 El sistema debe ser capaz de ejecutarse en diferentes plataformas (sistemas operativos, hardware, bases de datos) con mínimas modificaciones
Mantenibilidad
RNF08- Documentación Debe existir una documentación detallada y actualizada del código, arquitectura y procesos del sistema, que facilite la comprensión y el mantenimiento por parte del equipo técnico.
Soportabilidad y operatividad
RNF09- documentación de usuario Existirá una documentación clara y concisa para los usuarios, que les permita resolver problemas comunes y realizar tareas
